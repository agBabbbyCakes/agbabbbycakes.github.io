ERC721 NFT Tutorial

A Step by Step Guide of how to get your NFT on the Open Sea

Official | Author | Chris Cao | Works with Ape .v4

This is where the main content will go Feel free to copy and paste or do some other wizardy NFT to OpenSea

The hardest part is over. You have made the contract and understand the EIP-721. You are ready to deploy it on a testnet and see it for real on OpenSea. There are many different way to reach this end goal. I am going to show you an simple way to do it with a couple of tool outside ApeWorX. The Web3 ecosystem is incredible, so let me show off some cool tools to bring it all together.

9 Steps to the finish line: Create a metadata file and an Image for your NFT Host the 2 files on a NFT media manager site (optional) Save the new ipfs CID to the your contract. Compile new content in the contract Open Ape Console with the testnet and provider Mint NFT Verify on OpenSea ??? Profit

Creating a JSON meta data file

>Creating a JSON meta data file: NFT are a digital signature to represent ownership of something tangible. The most common use case is to proof of ownership of a picture and it details behind it. The details can be represented in a metadata file: The metadata file standard is defined by this standard EIP-1155

Host the 2 files on a NFT media manager site (optional)

One of the tool, I recommend is pinata. It is a Media Manager company for content creators and developers to integrate with IPFS (InterPlanetary File System)

IPFS is a protocol, hypermedia and file sharing peer to peer (p2p) for storing and sharing data in a distributed file system. We will be using pinata to manage a metadata file and its image to pin it on an ipfs node for all to use and see.

Save the new IPFS CID to your contract

3. Save the new IPFS CID to your contract CID means Content Identifier. When you host a file on ipfs, it hashes content address so you can use it discreetly. This is a sample of a CID

4. Compile the contract

With the new CID, make sure you find and replace all the CID references. You can test to make sure you got them all by running ape test.

Open Ape Console with testnet and provider

This is homestretch. Deploy the contract on a testnet, if you want it on OpenSea, you can deploy it on Rinkeby and Polygon if you like.

Mint the NFT

Verify on a marketplace (Open Sea)

Now that the contract is deployed and you have minted an NFT. Make sure you check it out and see your hard work! You can check your work out on the testnet linked to OpenSea and another testnet with Rariable too.
