Purpose

This guide is crafted to empower our developers to effectively utilize the Silverback Design System by providing clear instructions and examples on how to implement UI components and widgets. The system is built with flexibility in mind, allowing for easy adaptation and extension as our project requirements evolve.

Key Features

  • Modular CSS: Utilizing BEM practices, our CSS is organized into logical blocks, making it easy to reuse and maintain components across different parts of our applications without conflicts.
  • Integration with HTMX: Designed to work seamlessly with HTMX, our system facilitates adding dynamic behaviors to our applications without writing much JavaScript
  • Responsive and Accessible: Ensuring that our applications are usable on a variety of devices and accessible to all users, including those with disabilities.
  • Extensive Documentation: Each component, from buttons to panels, is documented with code snippets and usage guidelines to provide a comprehensive understanding for all team members.

Current Components

Typography settings (Headings, Paragraphs, Links)

UI Elements (Buttons, Inputs, Toggles, Dropdowns)

Layout principles (Spacing, Panels, Grid Systems)

Color System (Functional colors like alerts, success, info)

Future Inclusions

Expanded Component Library: We aim to broaden our array of components to include more complex UI elements such as sliders, progress bars, and modals.

Theming Capabilities: Plans to introduce theme support to allow easy switching between light and dark modes or customizing the application according to brand guidelines.

Performance Optimization: Further enhancements to ensure that our design system does not adversely impact the load times and interactivity of our applications.

Community Feedback Integration: Incorporating feedback from our users and developers to continuously improve the design system.

Conclusion

The Silverback Design System is more than just a style guide—it's a comprehensive toolset that empowers our team to create cohesive, robust, and scalable applications. As we continue to develop and refine this system, we remain committed to providing our team with the resources they need to deliver exceptional digital experiences.

Content Block Example

This section explains the CSS tags used to style the content blocks.

The .container-content class is used to center the content both horizontally and vertically. It ensures that the content takes up the full viewport height.

  • width: 100%; - Ensures the container takes the full width of the viewport.
  • display: flex; - Uses flexbox for layout.
  • flex-direction: column; - Arranges children in a column.
  • align-items: center; - Centers children horizontally.
  • justify-content: center; - Centers children vertically.
  • min-height: 100vh; - Ensures the container takes the full viewport height.

Content Block Example 2

The .content-block class is used to create a glass-like effect for the content blocks.

  • width: 80%; - Sets the width of the content block to 80% of its parent.
  • max-width: 800px; - Sets a maximum width for the content block.
  • background: rgba(255, 255, 255, 0.1); - Sets a semi-transparent background.
  • border-radius: 10px; - Rounds the corners of the content block.
  • padding: 20px; - Adds padding inside the content block.
  • box-shadow: 0 4px 6px rgba(0, 0, 0, 0.1); - Adds a subtle shadow.
  • backdrop-filter: blur(10px); - Applies a blur effect to the background.
  • -webkit-backdrop-filter: blur(10px); - Applies the blur effect for Safari.
  • margin: 20px 0; - Adds margin between content blocks.
  • color: var(--text-primary); - Sets the text color to ensure readability.

Content Block Example 3

This example demonstrates a content block with a minimum height of 50vh.

  • width: 90%; - Sets the width of the content block to 90% of its parent.
  • min-height: 50vh; - Ensures the content block takes at least half the viewport height.
  • background: rgba(0, 0, 0, 0.1); - Sets a semi-transparent background.
  • border-radius: 15px; - Rounds the corners of the content block.
  • padding: 30px; - Adds padding inside the content block.
  • box-shadow: 0 4px 8px rgba(0, 0, 0, 0.2); - Adds a more pronounced shadow.
  • backdrop-filter: blur(5px); - Applies a blur effect to the background.
  • -webkit-backdrop-filter: blur(5px); - Applies the blur effect for Safari.
  • margin: 30px 0; - Adds margin between content blocks.
  • color: var(--text-secondary); - Sets the text color to a secondary color.

Content Block Example 4

This example demonstrates a content block with a minimum height of 75vh.

  • width: 70%; - Sets the width of the content block to 70% of its parent.
  • min-height: 75vh; - Ensures the content block takes at least three-quarters of the viewport height.
  • background: rgba(255, 255, 255, 0.2); - Sets a semi-transparent background.
  • border-radius: 20px; - Rounds the corners of the content block.
  • padding: 40px; - Adds padding inside the content block.
  • box-shadow: 0 6px 10px rgba(0, 0, 0, 0.3); - Adds a more pronounced shadow.
  • backdrop-filter: blur(15px); - Applies a stronger blur effect to the background.
  • -webkit-backdrop-filter: blur(15px); - Applies the blur effect for Safari.
  • margin: 40px 0; - Adds margin between content blocks.
  • color: var(--text-tertiary); - Sets the text color to a tertiary color.

Spacing

Guidelines for numbers to use for padding and spacing. Josh's suggestion is 8px multipliers:

  • xxxs: 2px
  • xxs: 4px
  • xs: 8px
  • s: 12px
  • md: 16px
  • lg: 24px
  • xl: 32px
  • xxl: 40px
  • xxxl: 48px
  • xxxxl: 64px

This doesn't mean literally everything uses these values, but it gives a solid set of spacers to use in the majority of cases.
